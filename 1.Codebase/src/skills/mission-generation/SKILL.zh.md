# 任務生成規則

## 概述

請生成一個新的任務場景，建立本回合的敘事主軸。每個任務都應該呈現黑色幽默：看似成功，實際卻讓末日更接近。

## 生成要求

### 內容結構
1. 場景描述（350-500 字）
2. 任務目標
3. 主要困境或挑戰

### 語氣要求
- 黑色幽默
- 反諷
- 嘲諷強迫樂觀與制度性正能量

### 寫作風格限制 — 嚴格禁止
在任何敘事文字中，禁止使用以下類型的語言：
1. **自創比喻性複合詞**：禁止發明新的比喻詞彙來描述角色或心理狀態（例如「邏輯鎧甲」、「無菌室裡的生存策略」、「六邊形戰士」）。請直接描述。
2. **以科技/系統比喻心理狀態**：禁止將理性、情緒或個性比擬為電腦、防火牆、作業系統、工廠或免疫系統。
3. **極端英雄/戰士比喻**：禁止用戰士或殉道者隱喻描述道德或情感掙扎（例如「孤勇的正義鬥士」、「泥漿中的戰士」、「吹響號角」、「降維打擊」）。
4. **象徵物件對比組合**：禁止用成對的抽象物件來對比價值觀（例如「鏡子與濾鏡」、「手術刀與止痛藥」）。
5. **過度戲劇化的自我描述**：禁止將角色描繪為帶有繁複自我辯護的悲劇英雄（例如「被迫戴上壞人面具的英雄」、「笨拙地傷害了他人的有良知的好人」）。
請直接、具體地描述角色、情緒與處境，不可發明詩意比喻複合詞。

### 細節要求
- 描述環境、角色互動與氛圍
- 保持 Gloria / Donkey / ARK / One 的角色一致性
- 任務中要出現「成功也是失敗」的感覺

## Choice Preview

在 `story_text` 最後加入 3 到 5 行選項預告，並與 JSON choices 對應：
- `[謹慎]`
- `[權衡]`
- `[瘋狂]`
- `[樂觀]`
- `[抱怨]`

每行需預告選項後續走向。

## 輸出限制

- 嚴格遵守外層 prompt 指定的 JSON schema
- 不可翻譯 canonical JSON key 與 archetype id
- 若格式錯誤，視為無效回應
