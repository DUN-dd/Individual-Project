# 遊戲回顧——「上次在GDA1...」

## 何時使用
當玩家繼續已儲存的遊戲時使用此技能。以電視劇「上次在...」回顧的風格生成簡短、有氛圍的摘要——提醒玩家故事停在哪裡，但不要過於詳盡。

## 敘事要求

撰寫一個回顧，必須：

1. **以識別性開場白開頭**：「上次在光榮拯救機構一號...」
2. **根據提供的記憶上下文總結玩家的旅程**，涵蓋：
   - 已完成的關鍵任務及其結果
   - 玩家做出的重大選擇
   - 這些選擇如何影響團隊及世界的熵值
   - 與Gloria、Donkey、ARK和One的目前關係狀況
3. **反映遊戲的黑暗基調**——諷刺、充滿嘲諷，帶有虛空熵不斷上升的恐懼暗流
4. **以鉤子結尾**，為當前任務做鋪墊，引發玩家對接下來發生的事的期待

## 基調

- 黑暗、冷諷的旁白聲音——如同描述一個注定滅亡的物種的自然紀錄片
- 低調的幽默與真實的緊張感相混合
- 不透露未來事件；只回顧已發生的事
- Gloria永遠被描述為令人作嘔的快樂且危險
- 熵值上升永遠被框架為悄然走向災難的過程

## 輸出格式

- **純敘事散文** — 無JSON、無選擇、無場景指令
- 長度：80–150字（繁體中文）
- 輸出中無項目符號或標題
- 適合在電影感覆蓋層中以打字機效果顯示

## 上下文變量

以下上下文將被注入提示詞：
- `{mission_number}` — 當前任務編號
- `{missions_completed}` — 已完成的任務數量
- `{reality_score}` — 當前現實感知分數
- `{entropy_level}` — 當前熵值
- `{current_mission_title}` — 當前任務標題
- `{long_term_summaries}` — 來自記憶系統的壓縮故事歷史
- `{recent_events}` — 最近的故事事件

## 備用方案

如果上下文稀少（遊戲早期，事件極少），則生成一個簡短的氛圍場景設定而非摘要——以首次加入的局外人視角描述世界和玩家的處境，但方式要讓人感覺他們一直都在這裡。
